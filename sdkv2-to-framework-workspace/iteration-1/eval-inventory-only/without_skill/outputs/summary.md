# Migration Summary

The terraform-provider-openstack is a large, exclusively SDKv2 provider (v2.38.1) with 109 managed resources and 64 data sources spanning 18 service areas and approximately 52,000 lines of non-test Go, with zero existing terraform-plugin-framework code. The migration checklist is structured in five phases: Phase 0 adds framework dependencies and a mux server so both SDKv2 and framework resources coexist; Phase 1 pilots six simple Tier 1 resources to prove the pattern; Phase 2 migrates all 64 data sources; Phase 3 migrates the 80+ Tier 2 moderate-complexity resources service-by-service; Phase 4 tackles the 20+ high-complexity Tier 3 resources including `openstack_compute_instance_v2` (1758 lines, CustomizeDiff, StateFunc, custom hash functions) and the entire networking and load-balancer service areas; Phase 5 removes SDKv2 entirely and cleans up the entry point. Key cross-cutting risks are TypeSet custom hash function removal (requiring state migration in 5 files), 63 files using `ValidateFunc` that need framework validator equivalents, and shared SDK-coupled helper files (`util.go`, `lb_v2_shared.go`, `networking_*_shared.go`) that must be refactored before their consumer resources can be migrated.
