# Migration summary: terraform-provider-openstack/v3

The OpenStack provider (SDKv2 v2.38.1, module `terraform-provider-openstack/v3`) is a large-scale migration: 236 Go source files, 109 registered resources and 64 data sources, all requiring conversion from `github.com/hashicorp/terraform-plugin-sdk/v2` to `terraform-plugin-framework`. The semgrep audit identifies 637 `ForceNew` occurrences (the dominant mechanical change), 140 validators, 101 importers, 68 timeout blocks, 11 MaxItems:1 nested-block candidates requiring a block-vs-single-nested-attribute decision (recommend backward-compatible `ListNestedBlock` for all, given no major version bump), 3 `CustomizeDiff` handlers (must become `ModifyPlan`), 4 `StateFunc` uses (must become custom types), 8 `DiffSuppressFunc` entries requiring per-case intent analysis, 1 `StateUpgraders` entry (`openstack_objectstorage_container_v1` — the sole SchemaVersion:1 resource, requiring a single-step `ResourceWithUpgradeState` implementation), and 20 `Sensitive: true` attributes. The checklist organises the 173 items (109 resources + 64 data sources) into four complexity tiers: Tier 1 (simple, no nested blocks — pilot with identity/segment resources and availability-zone data sources), Tier 2 (timeouts + importers), Tier 3 (nested blocks, DiffSuppressFunc, CustomizeDiff), and Tier 4 (the state-upgrader resource); `openstack_compute_instance_v2` is flagged as the highest-complexity single item and should be migrated last or treated as a mini-project of its own.
