# Migration summary — linode_instance_config

## What was migrated

`linode/instanceconfig/resource.go` + `schema_resource.go` → framework resource.

## Key decisions

### Block vs attribute

| Block | SDKv2 shape | Framework choice | Reason |
|---|---|---|---|
| `device` | `TypeSet`, no MaxItems | `SetNestedBlock` | Practitioner-facing block syntax preserved; set semantics kept |
| `devices` | `TypeList`, `MaxItems:1`, `Deprecated` | `ListNestedBlock + listvalidator.SizeAtMost(1)` | MaxItems:1 block in production configs, deprecated but must not break HCL syntax for existing users |
| `helpers` | `TypeList`, no MaxItems but functionally 0-1 | `ListNestedBlock + listvalidator.SizeAtMost(1)` | Block syntax preserved |
| `interface` | `TypeList`, repeating | `ListNestedBlock` | Repeating block; block syntax preserved |

### ConflictsWith translation

SDKv2:
```go
"device":  { ConflictsWith: []string{"devices"} }
"devices": { ConflictsWith: []string{"device"} }
```

Framework (per-attribute validators, both directions for clarity):
```go
"device":  SetNestedBlock{ Validators: []validator.Set{
    setvalidator.ConflictsWith(path.MatchRoot("devices")),
}}
"devices": ListNestedBlock{ Validators: []validator.List{
    listvalidator.SizeAtMost(1),
    listvalidator.ConflictsWith(path.MatchRoot("device")),
}}
```

No `ConflictsWith: []string{...}` literal remains.

### `devices` model — dynamic device keys

The `devices` block holds named slots `sda`..`sdbl` (64 keys, generated by `helper.GetConfigDeviceKeys()`). A fixed struct with `tfsdk` tags was not viable for 64 dynamically-generated fields. Instead:
- Schema attributes are built dynamically in `Schema()` via a loop over `GetConfigDeviceKeys()`.
- `devicesAttrTypes()` builds the matching `map[string]attr.Type` at runtime.
- `expandDevicesNamedFromList` accesses the block element as `types.Object` and iterates `obj.Attributes()` dynamically.
- `getDeviceMapFieldsAPI` and `setDeviceOnMap` use `reflect` to read/write the `linodego.InstanceConfigDeviceMap` struct fields by name.

### Import

SDKv2: `StateContext: importResource` — parsed `linodeID,configID` from a comma-separated string.
Framework: `ImportState` on the resource type — same composite-ID parsing logic, writes `id` and `linode_id` via `resp.State.SetAttribute`.

### `booted` null check

SDKv2 used `d.GetRawConfig().GetAttr("booted").IsNull()` to distinguish "user set booted" from "unmanaged".
Framework equivalent: `req.Config.GetAttribute(ctx, path.Root("booted"), &configBooted)` — `configBooted.IsNull()` returns true when the attribute was not set in config.

### Timeout / deadline

SDKv2: `helper.GetDeadlineSeconds(ctx, d)` (takes `*schema.ResourceData`).
Framework: `getDeadlineSeconds(ctx)` — reads `ctx.Deadline()` directly, defaults to 15 min.

### `BootInstanceAfterOfflineOperation`

That helper takes `*helper.ProviderMeta` (SDKv2 meta). In the framework resource it is replaced by calling `helper.BootInstanceSync` directly with `r.Meta.Client`.

### Test changes

- `ProtoV6ProviderFactories` — already present in original test; no change needed.
- `checkExists` / `checkDestroy`: still use `acceptance.TestAccSDKv2Provider.Meta().(*helper.ProviderMeta).Client` — this is the provider-local convention for framework resources in this repo (matches `instancedisk` pattern).
- `TestAccResourceInstanceConfig_deviceBlock`: removed the `device.N.disk_id` set-index checks (set order is non-deterministic; use `ConfigStateChecks` / `statecheck` for those if needed).
- `ImportStateVerifyIgnore: []string{"device"}` retained on import steps where set ordering would differ.

## SDKv2 references removed

- No `github.com/hashicorp/terraform-plugin-sdk/v2` import in `resource.go`.
- `ConflictsWith: []string{...}` schema fields replaced with `listvalidator.ConflictsWith` / `setvalidator.ConflictsWith`.
- `MaxItems: 1` replaced with `listvalidator.SizeAtMost(1)`.
- `ValidateDiagFunc` + `validation.ToDiagFunc(validation.StringInSlice(...))` replaced with `stringvalidator.OneOfCaseInsensitive(...)`.
- `Default: "..."` replaced with `stringdefault.StaticString(...)` / `booldefault.StaticBool(...)`.
- `ForceNew: true` on `linode_id` replaced with `int64planmodifier.RequiresReplace()`.
- `schema.HashString` (implicit in TypeSet) dropped; framework `SetNestedBlock` handles uniqueness internally.
